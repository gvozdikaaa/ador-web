# Веб-приложение «Подростки о родителях» (ADOR)

Веб-приложение для диагностики детско-родительских отношений по методике
ADOR (Шафер, в модификации З. Матейчика и П. Ржичана).

## Стек технологий

Операционная система разработки — Windows 11. Среда разработки — Visual Studio
Code. Язык программирования — JavaScript. Серверная платформа — Node.js
версии 18 или новее с веб-фреймворком Express 4. Шаблонизатор представлений —
EJS. Стили оформления реализованы средствами CSS без препроцессоров. Хранение
данных обеспечивает СУБД MySQL 8. Отправка писем педагогу-психологу
выполняется через Nodemailer по протоколу SMTP.

## Установка и первый запуск

Установка зависимостей выполняется командой `npm install` в каталоге
проекта. Для создания базы данных следует запустить MySQL Workbench
или консольный клиент и выполнить скрипт `sql/schema.sql` под учётной
записью с правами создания базы. Скрипт создаёт базу `ador_db`, все
необходимые таблицы и заполняет справочник из 50 утверждений теста,
а также добавляет начальную учётную запись администратора (логин
`admin`, пароль `admin123`). После первого входа пароль рекомендуется
сменить через панель администратора.

Перед запуском сервера необходимо скопировать `.env.example` в `.env`
и указать параметры подключения к MySQL и SMTP-серверу для отправки
писем. Запуск сервера осуществляется командой `npm start`. Приложение
открывается по адресу `http://localhost:3000`. Для разработки удобнее
использовать команду `npm run dev`, обеспечивающую автоматический
перезапуск сервера при изменении файлов через `nodemon`.

## Настройка отправки почты

Для отправки писем через Gmail требуется использовать пароль
приложения, а не основной пароль аккаунта. Получить его можно в
настройках безопасности учётной записи Google по ссылке
`https://myaccount.google.com/apppasswords` (для этого на аккаунте
должна быть включена двухэтапная аутентификация). Полученный
шестнадцатизначный пароль приложения помещается в переменную
`SMTP_PASS` файла `.env`, при этом адрес отправителя указывается в
переменной `SMTP_USER`.

Google показывает пароль приложения четырьмя группами по четыре
символа, разделёнными пробелами (например, «abcd efgh ijkl mnop»).
Приложение автоматически удаляет пробелы из `SMTP_PASS`, поэтому
пароль можно вставлять в любом виде.

Если письма не доходят, несмотря на корректную, как кажется,
настройку, следует проверить следующее. Во-первых, соответствие порта
и режима шифрования: для порта `587` переменная `SMTP_SECURE` должна
быть равна `false` (используется STARTTLS), а для порта `465` —
`true` (используется SSL); приложение приводит этот параметр в
соответствие порту автоматически. Во-вторых, используется именно
пароль приложения, а не обычный пароль аккаунта. В-третьих, исходящий
порт SMTP не заблокирован сетью или брандмауэром учреждения. При
запуске сервера в консоль выводится результат проверки SMTP-соединения
и, в случае ошибки, её код — это помогает быстро определить причину.

Адрес электронной почты педагога-психолога — получателя результатов —
по умолчанию равен `psychologist.mrk@gmail.com` и хранится в таблице
`settings` базы данных. Изменить его можно через панель
администратора в разделе «Настройки системы» без перезапуска сервера.

## Использование

Подросток открывает страницу приложения, вводит фамилию, имя, номер
учебной группы, возраст и пол. После прочтения инструкции он
последовательно заполняет два бланка по 50 утверждений: сначала
в отношении матери, затем в отношении отца. Каждое утверждение
оценивается по трёхбалльной шкале (2 — «Да», 1 — «Частично»,
0 — «Нет»). По завершении тестирования подростку отображаются
численные значения баллов по пяти шкалам отдельно для матери и для
отца, сопровождаемые мотивирующим сообщением и рекомендацией
обратиться к педагогу-психологу за подробной интерпретацией. Полный
отчёт автоматически направляется на почту педагога-психолога.

Администратор входит в систему по ссылке «Вход для администратора»
в шапке сайта или напрямую по адресу `/admin/login`. В панели ему
доступны просмотр списка прошедших тест и подробных результатов
каждого пользователя, удаление записей, редактирование текста
утверждений теста, настройка адреса педагога-психолога, текста
инструкции и мотивирующего сообщения, а также смена пароля.

## Описание методики

Опросник ADOR изучает установки, поведение и методы воспитания
родителей в восприятии подростка. Он состоит из 50 утверждений,
оценивающих пять шкал по 10 пунктов: позитивный интерес (POZ) —
утверждения 1, 6, 11, 16, 21, 26, 31, 36, 41, 46; директивность (DIR) —
утверждения 2, 7, 12, 17, 22, 27, 32, 37, 42, 47; враждебность (HOS) —
утверждения 3, 8, 13, 18, 23, 28, 33, 38, 43, 48; автономность (AUT) —
утверждения 4, 9, 14, 19, 24, 29, 34, 39, 44, 49; непоследовательность
(NED) — утверждения 5, 10, 15, 20, 25, 30, 35, 40, 45, 50.

Каждое утверждение оценивается в баллах: 2 балла («Да»), 1 балл
(«Частично»), 0 баллов («Нет»). Сумма по шкале (диапазон от 0 до 20)
переводится в стандартизированный балл в диапазоне от 1 до 5 по
нормативной таблице. Норма соответствует значению 3. Значения 1–2
означают слабую выраженность качества, значения 4–5 — отчётливое
проявление.

В файле `utils/scoring.js` приведены упрощённые нормативные таблицы
перевода сырых баллов в стандартизированные. При необходимости их
можно заменить на точные таблицы из научной публикации
Вассермана Л. И., Горьковой И. А. и Ромицыной Е. Е., поскольку код
поддерживает разные таблицы для каждого сочетания пола подростка
и оцениваемого родителя.
